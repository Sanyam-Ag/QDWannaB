# v0.3 DESIGN PLANS

## Motivation & Aim

> To implement file loading allowing the engine to work on data derived from the filesystem instead of the data inserted before the code compilation in the code itself, though, no NaN or missing value handling for this version intial version
> File loading support for only numerical fields in this version
> Improved parsing and more dynamic expression support. Eg: add col1 col2 newcol, etc
> Time aware statistics and analysis, Rolling window support(later mini version)

## v0.3 Scope

1. File I/O
2. Dynamic Parsing
3. Rolling Stats
4. **This version only supports 1 dataset in memory at a perticular session instance**

## Future Scope

1. Dynamic File I/O
2. Networking Interface
3. MultiLanguage Bindings
4. Visualisation
5. Caching
